# 🎓 Faculty Guide: Managing Assessments

Welcome to the Placement Assessment Platform. This guide will help you create, manage, and monitor coding exams for your students.

## 1. Onboarding Students
Before running an assessment, ensure your students have accounts.
- Go to the **Users** tab in the navigation bar.
- Use the **Bulk Import** feature.
- You can upload an **Excel (.xlsx)** or **CSV** file with student names and emails.
- Set a **Default Password** (e.g., `College@2026`) that students will use for their first login.
- **Tip:** If a student forgets their password, find them in the **User List** and click **Reset Password**.

## 2. Creating an Assessment
- Go to **Manage Assessments** -> **Create Assessment**.
- **Title & Description:** Provide clear instructions for the students.
- **Schedule:** Set a strict Start and End time. Students cannot start before the start time or submit after the end time.
- **Duration:** This is the time (in minutes) a student has once they click "Start".
- **Problem Selection:** Add problems from the question bank and assign maximum marks for each.

## 3. Reviewing Before Launch
- Always use the **Preview** button on your assessment dashboard.
- This allows you to see the configuration exactly as students will, including the list of problems and total marks.
- Click **View Student Landing** to see the exact page students will see before they begin.

## 4. Live Monitoring (During the Exam)
- Once the exam is live, go to the **Results** page for that assessment.
- You will see the **LIVE MONITORING** dashboard.
- Track student statuses in real-time:
  - **Not Started:** Student hasn't logged in yet.
  - **Active:** Student is currently coding.
  - **Submitted:** Student finished early.
  - **Timed Out:** System automatically finished the attempt when time expired.
- The dashboard auto-refreshes every 30 seconds.

## 5. Exporting Results
- After the exam, click **Export to Excel** on the Results page.
- This will download a full report including student details, final scores, and exact start/end timestamps.
